# 部署指南：使用 1Panel 部署 React 静态网站

## 第一步：获取构建产物（已在本地完成）
我已经帮你运行了 `npm run build`，现在的最终网页文件都在你项目目录下的 `dist` 文件夹里。
目录路径：`D:\key\dist`

你需要上传的就是这个 `dist` 文件夹里的**所有内容**（index.html, assets文件夹等）。

## 第二步：在 1Panel 面板创建网站
1. 登录你的 1Panel 管理面板。
2. 点击左侧菜单的 **"网站"** -> **"网站"**。
3. 点击 **"创建网站"** 按钮。
4. 在弹出的窗口中配置：
   - **运行环境**：选择 `静态站点` (OpenResty/Nginx)。
   - **主域名**：填写你的域名（例如 `guiguisocute.cloud`）。
   - **代号/目录**：会自动生成，通常默认即可。
   - 点击 **"确认"**。

## 第三步：上传文件
1. 在网站列表中，找到你刚才创建的网站。
2. 点击网站行中的 **"文件"** 或 **"根目录"** 链接，进入文件管理界面。
3. **删除** 目录下默认生成的 `index.html` 或 `404.html`（如果有）。
4. 点击 **"上传"** -> **"上传目录"**（如果支持）或者将 `dist` 文件夹里的内容打包成 `dist.zip` 上传后解压。
   - **重点**：确保 `index.html` 文件直接位于网站根目录下，**不要**多嵌套一层 `dist` 文件夹。
   - 正确结构示例：
     ```
     /www/sites/guiguisocute.cloud/index.html
     /www/sites/guiguisocute.cloud/assets/
     ```

## 第四步：配置 Nginx (SPA 刷新支持)
虽然目前的单页应用主要用锚点（#），但为了最佳实践和防止未来加路由报错，建议配置一下伪静态。

1. 在网站列表中，点击你的网站名称进入 **"配置"**。
2. 点击 **"配置文件"** 标签页（或者 "伪静态" / "URL重写"）。
3. 在 `location / { ... }` 块中，确保有以下配置（如果是 1Panel 的伪静态设置，选择 `location /` 规则）：

```nginx
location / {
    try_files $uri $uri/ /index.html;
    index index.html index.htm;
}
```
*（注：如果是纯静态站点且没有使用 React Router 的 path 模式，只用 # 跳转，这一步其实可以跳过，默认配置即可工作）*

## 第五步：开启 HTTPS (推荐)
1. 在网站配置中点击 **"HTTPS"**。
2. 如果你有证书，选择上传；或者选择 **"Acme 账户"** 申请免费的 Let's Encrypt 证书。
3. 勾选 **"开启 HTTPS"** 和 **"强制 HTTPS"**。

---
完成以上步骤，你的网站就上线了！
